# 🐍 Aula 1 — Introdução ao Python e criação do primeiro módulo (PyCharm)

## 🎯 Objetivo da aula
Começar o estudo de Python do zero, organizando corretamente o ambiente de trabalho e aprendendo a criar o primeiro arquivo Python.

A aula reforça:
- Organização do projeto
- Boas práticas de nomes de pastas e arquivos
- Criação do primeiro arquivo `.py`
- Entendimento do que é um módulo Python

---

## 🧱 Conceitos iniciais importantes

Antes de programar, precisamos organizar corretamente nosso projeto.

Em Python:
- Cada arquivo `.py` é chamado de **módulo**.
- Projetos bem organizados evitam problemas futuros.
- Nomeação correta evita erros no sistema e no Python.

---

## 📁 Criando o projeto no PyCharm

### Passo a passo no PyCharm

1. Abra o **PyCharm**
2. Clique em **New Project**
3. Escolha onde salvar o projeto (ex: Desktop ou pasta de estudos)
4. Nomeie o projeto seguindo boas práticas: ex: `curso_python`
5. Clique em **Create**

O PyCharm cria automaticamente:
- Pasta do projeto
- Ambiente Python (virtual environment)
- Estrutura básica

---

## 📝 Criando o primeiro arquivo Python

Dentro do PyCharm:

1. Clique com botão direito na pasta do projeto
2. Selecione: New → Python File
3. Nomeie o arquivo: `aula_01`

## 🧠 Regras importantes para nomes de pastas e arquivos

⚠️ Evite erros usando estas regras:

### ✅ Permitido
- Letras: `a-z`
- Números
- Underline `_`

### ❌ Evitar
Não use:
- Espaços
- Acentos
- Caracteres especiais
- Começar com número

## 📌 Por que evitar espaços e acentos?

Porque:
- Alguns sistemas não lidam bem com caracteres especiais
- Importações podem falhar
- URLs e caminhos podem quebrar
- Projetos profissionais evitam isso
---
## ✅ Resumo da aula
- Python organiza o código em arquivos chamados módulos (.py).
- Projetos devem ser organizados desde o início.
- O PyCharm pode criar automaticamente a estrutura inicial do projeto.
- Arquivos Python são criados dentro da pasta do projeto.
- Nomes de pastas e arquivos devem evitar espaços e caracteres especiais.
- Utilizar apenas letras, números e _ evita problemas futuros.
- Boa organização facilita manutenção e evolução do projeto.
